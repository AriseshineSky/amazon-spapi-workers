# 公开仓库说明

本仓库可安全公开：**凭证不在代码里**，只放在服务器本地的 `config.ini` 和环境变量中。

## 发布前已处理

| 项目 | 说明 |
|------|------|
| Telegram Bot | 已改为 `[telegram]` 配置或 `TELEGRAM_BOT_TOKEN` 环境变量 |
| Amazon / ES / Redis | 仅出现在 `config.ini.sample`、`.env.example` 的占位符 |
| Git 历史 | **首次公开推送使用全新历史**（不含旧 monorepo 里可能泄露的 token） |

## 不要提交的文件

- `deploy/deploy.env`（含 Redis 密码）
- `~/.amazon_spapi/config.ini`
- `.env`

## 克隆与部署

```bash
git clone https://github.com/AriseshineSky/amazon-spapi-workers.git ~/src/em-workers
cd ~/src/em-workers
cp deploy/deploy.env.example deploy/deploy.env   # 编辑 BROKER_URL
cp config.ini.sample ~/.amazon_spapi/config.ini  # 编辑凭证
sudo ./deploy/install-ubuntu.sh
```

详细步骤：[生产部署.md](./生产部署.md)

## 服务器自检（无需跑 Celery 任务）

```bash
export BROKER_URL='redis://...'
export AMAZON_SPAPI_CONFIG_PATH=~/.amazon_spapi/config.ini
./deploy/check-server.sh

# Worker 已启动时，额外检测 Celery 进程
./deploy/check-server.sh --with-workers

# 检查 systemd 服务
./deploy/check-server.sh --systemd amazon-spapi-offers-us.service
```

全部 `[OK]` 且 exit code 0 表示环境就绪；**不会**向 Amazon 发真实 API 请求。
